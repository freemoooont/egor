# Aggregates

For each aggregate: root entity, child entities, value objects, **numbered, testable**
invariants, transactional boundary, lifecycle states, identifier rules, and the
identifying domain events. Each invariant is phrased so a single Go test can assert it
in isolation — these names will become the test names during TDD.

---

## `iam.User`

**Root entity.** `User`.

**Child entities.**
- `RefreshTokenFamily` — a chain of `RefreshToken` records belonging to a single
  login session, identified by `family_id` (UUID).

**Value objects.**
- `EmailAddress` — RFC-5321 valid, lower-cased, ≤254 chars.
- `PasswordHash` — bcrypt cost ≥10. Plaintext never crosses the domain boundary.
- `DisplayName` — 1..64 graphemes after Unicode trim.
- `AvatarRef` — `none | server-ref<string>`.

**Identifier.** UUID v4 generated by the `IDGenerator` port at `RegisterUser` time.
Immutable.

**Invariants (testable in isolation).**
1. `User_EmailAddressMustBeNonEmptyAfterTrim` — building a `User` with an empty/whitespace email returns `ErrInvalidEmail`.
2. `User_EmailAddressMustBeRFC5321Valid` — emails failing the validator return `ErrInvalidEmail`.
3. `User_EmailAddressIsLowerCasedOnConstruction` — `"Foo@Bar.COM"` becomes `"foo@bar.com"`.
4. `User_EmailAddressIsUniqueAcrossUsers` — `Users.Save` of a second user with the same email returns `ErrEmailTaken`.
5. `User_PasswordHashMustComeFromBcryptHasher` — constructing a `User` from a plaintext-looking string panics in dev / returns `ErrInvalidPasswordHash` in prod.
6. `User_DisplayNameLengthBetween1And64` — names of length 0 or >64 return `ErrInvalidDisplayName`.
7. `User_ChangePasswordRotatesHashAndRevokesAllRefreshFamilies` — after `ChangePassword` every active `RefreshTokenFamily` for the user is revoked.
8. `RefreshTokenFamily_TokensAreOrderedByIssuedAt` — within a family, `issued_at` is strictly increasing.
9. `RefreshTokenFamily_OnlyTheLatestTokenIsValid` — presenting any non-latest token to `RefreshAccessToken` triggers `RevokeFamily` and returns `ErrRefreshTokenReused`.
10. `RefreshTokenFamily_ExpiredTokenCannotMint` — a token past its 7-day TTL returns `ErrRefreshTokenExpired`.
11. `User_AvatarRefDefaultsToNoneOnRegistration` — fresh users have `AvatarRef = none`.

**Transactional boundary.** A single `pgx.Tx` per use case. `RegisterUser` writes the
user row + the initial `RefreshTokenFamily` + the initial `RefreshToken` + the
`OutboxRecord` (one per emitted `DomainEvent`) atomically. `RefreshAccessToken` revokes the consumed token and inserts the
rotated token + outbox in one tx. The aggregate never spans more than one user.

**Lifecycle states.** `Active` (default). No soft-delete in v1 — account deletion is a
non-goal. Refresh-token families have `Active | Revoked`.

**Identifying events.**
- `UserRegistered` (creates the aggregate)
- `UserLoggedIn`
- `RefreshTokenIssued`
- `RefreshTokenRevoked`

## `decks.Deck`

**Root entity.** `Deck`.

**Child entities.**
- `Card` — entity inside the aggregate. Mutated **only** through `Deck` methods
  (`AddCard`, `EditCard`, `RemoveCard`, `ReorderCards`). No `cards` repository exists.

**Value objects.**
- `DeckTitle` — 1..120 graphemes after trim.
- `OwnerID` — UUID; immutable; matches an `iam.User.ID`.
- `Term` — 1..512 graphemes after trim.
- `Definition` — 1..2048 graphemes after trim.
- `CardOrdinal` — 1-based dense integer; unique within the deck.

**Identifier.** UUID v4 generated at `CreateDeck`. Card ids are also UUID v4.

**Invariants (testable in isolation).**
1. `Deck_TitleLengthBetween1And120` — empty/whitespace title returns `ErrInvalidDeckTitle`; 121-char title returns `ErrDeckTitleTooLong`.
2. `Deck_OwnerIDIsImmutable` — there is no `SetOwner` method; struct field is unexported.
3. `Deck_HasAtMost500Cards` — adding a 501st card returns `ErrDeckCardLimitExceeded`.
4. `Deck_CardOrdinalsAreDense1ToN` — at any point the multiset of `CardOrdinal`s equals `{1, 2, ..., N}` where `N = len(cards)`.
5. `Deck_CardOrdinalsAreUnique` — no two cards in the same deck share an ordinal.
6. `Card_TermLengthBetween1And512` — out-of-range term returns `ErrInvalidTerm`.
7. `Card_DefinitionLengthBetween1And2048` — out-of-range definition returns `ErrInvalidDefinition`.
8. `Deck_AddCardAppendsAtNextOrdinal` — `AddCard` assigns ordinal `N+1` where `N` is the current card count.
9. `Deck_RemoveCardRecompactsOrdinals` — after `RemoveCard`, ordinals are recompacted to `1..N-1` (no gaps).
10. `Deck_ReorderCardsRequiresExactPermutationOfExistingIDs` — passing a list missing or duplicating ids returns `ErrInvalidCardReorder`.
11. `Deck_RenameDoesNotMutateCards` — `Rename` only touches `DeckTitle`; cards are unchanged.
12. `Deck_DeleteIsTerminal` — after `Delete`, all subsequent mutators return `ErrDeckDeleted`.
13. `Deck_OnlyOwnerCanMutate` — every mutator that takes a caller `user_id` returns `ErrForbidden` when caller ≠ `OwnerID`.

**Transactional boundary.** One `pgx.Tx` per use case. A use case that adds three cards
in a row (e.g. AI-generated import, or `CreateDeck` with bundled cards) writes the
deck row + all card rows + outbox in a single tx. The aggregate boundary is exactly
one `Deck` plus its `Card`s.

**Lifecycle states.** `Active | Deleted`. Deletion is hard in `decks.cards` but the
`decks.decks` row is soft-deleted (`deleted_at timestamptz`) so emitted events still
have a foreign key target until the outbox publishes them.

**Identifying events.**
- `DeckCreated` (creates the aggregate)
- `DeckRenamed`
- `DeckDeleted`
- `CardAdded`
- `CardEdited`
- `CardRemoved`
- `CardsReordered`
- `DeckGenerationRequested` (emitted from the AI stub use case before the deck exists)
- `DeckGenerationCompleted` (emitted from the AI stub use case once a draft is ready)

## `practice.Session`

**Root entity.** `Session` (a.k.a. `PracticeSession`).

**Child entities.**
- `RatedCard` — at most one per `card_id` per session. Re-rating updates the existing
  entry rather than appending. Mutated only through `Session.Rate`.

**Value objects.**
- `PracticeSessionMode` — `Tracked | Untracked`.
- `PracticeSessionStatus` — `InProgress | Completed | Abandoned`.
- `CardRating` — `DontKnow=0 | StillLearning=1 | KnowKnow=2` (smallint).
- A snapshot list of `card_id`s captured at session start, used to bound which cards
  may be rated within this session.

**Identifier.** UUID v4 generated at `StartPracticeSession`.

**Invariants (testable in isolation).**
1. `Session_StatusStartsAsInProgress` — fresh sessions have `status = InProgress`.
2. `Session_RateRequiresInProgressStatus` — rating after `Finish` returns `ErrSessionClosed`.
3. `Session_RateRequiresTrackedMode` — rating in `Untracked` mode returns `ErrSessionUntracked`.
4. `Session_RateRejectsCardsNotInSnapshot` — rating a `card_id` not in the snapshot returns `ErrCardNotInSession`.
5. `Session_RateIsIdempotentPerCardID` — rating the same `card_id` twice keeps a single `RatedCard` whose value equals the latest call.
6. `Session_FinishIsTerminalAndIdempotent` — first `Finish` sets `status = Completed` and stamps `completed_at`; subsequent `Finish` calls return the same result.
7. `Session_AbandonIsTerminalAndIdempotent` — first `Abandon` sets `status = Abandoned`; subsequent calls are no-ops.
8. `Session_OwnedBySingleUser` — `user_id` is set at construction and immutable.
9. `Session_DeckIDIsImmutable` — `deck_id` is set at construction and immutable.
10. `Session_TrackedFinishEmitsCardRatedSummary` — `PracticeSessionCompleted` payload counts `DontKnow`/`StillLearning`/`KnowKnow` exactly as in the rated-card list.

**Transactional boundary.** One `pgx.Tx` per use case. `StartPracticeSession` writes
the session row + the card-id snapshot + outbox in one tx. `RateCard` writes/updates
the `RatedCard` row + outbox in one tx. `FinishPracticeSession` flips the status +
emits `PracticeSessionCompleted` to the outbox in one tx. The downstream
`UserDeckProgress` update is performed by the in-process event handler (see ADR 0002).

**Lifecycle states.** `InProgress → Completed` (success path) or `InProgress → Abandoned`
(via timeout cleanup or explicit cancel). Both terminal states are immutable.

**Identifying events.**
- `PracticeSessionStarted`
- `CardRated`
- `PracticeSessionCompleted`

## `practice.UserDeckProgress`

**Root entity.** `UserDeckProgress` — a read-model aggregate.

**Child entities.** None (a flat list of `CardProgress` value objects).

**Value objects.**
- `CardProgress` — `(card_id, latest CardRating, last_rated_at)`.

**Identifier.** Composite `(user_id, deck_id)`. No surrogate UUID — this is a read
model, not a write target from outside.

**Invariants (testable in isolation).**
1. `UserDeckProgress_OneRowPerUserDeckPair` — applying two `PracticeSessionCompleted` events for the same `(user_id, deck_id)` results in a single read-model row.
2. `UserDeckProgress_LatestRatingWins` — `CardProgress.rating` always reflects the most recent `Tracked` `PracticeSessionCompleted`.
3. `UserDeckProgress_UntrackedSessionsAreIgnored` — `Untracked` sessions never mutate the read model.
4. `UserDeckProgress_DropsCardOnCardRemoved` — consuming `CardRemoved` removes the matching `CardProgress`.
5. `UserDeckProgress_DropsAllOnDeckDeleted` — consuming `DeckDeleted` deletes the entire row.

**Transactional boundary.** Updated **only** by event handlers, in their own `pgx.Tx`,
using the corresponding outbox record's id as a deduplication key (see ADR 0002).

**Lifecycle states.** Implicit: a row exists once a tracked session has completed for
that `(user_id, deck_id)`; it disappears when its deck is deleted.

**Identifying events.** None — `UserDeckProgress` is a projection. It consumes
`PracticeSessionCompleted`, `CardRemoved`, `DeckDeleted`.
